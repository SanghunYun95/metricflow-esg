Go context.Done() channel closed behavior multiple goroutines

💡 Result:

In Go, the context.Done channel is safe for use by multiple goroutines. When a context is canceled (via its cancel function or timeout/deadline), the Done channel is closed once, and all goroutines selecting on or receiving from it will immediately detect the closure and proceed. Key behaviors: - The same Context instance (and its single Done channel) can be passed to multiple goroutines safely. - Closing happens asynchronously but is atomic; multiple goroutines calling cancel are safe—only the first has effect, subsequent calls do nothing. - Receiving from a closed Done channel always succeeds immediately, returning the zero value for struct{} (nil), allowing all waiting goroutines to exit promptly without race conditions or panics. - Goroutines typically use select statements to check ctx.Done alongside other operations, ensuring cooperative cancellation. Example pattern for multiple goroutines: ctx, cancel := context.WithCancel(context.Background) var wg sync.WaitGroup wg.Add(2) go func { defer wg.Done for { select { case <-ctx.Done: return default: // do work } } } go func { defer wg.Done // similar loop } go func { // some time later cancel // closes Done once; both goroutines detect it } wg.Wait This prevents goroutine leaks and enables graceful shutdowns. Never close ctx.Done yourself—it's managed internally by the context package.
